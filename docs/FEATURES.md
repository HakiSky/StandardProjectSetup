# Features And Project Knowledge

Use this file to record implemented systems, workflows, mechanics, and reusable
patterns. This is where Codex should look for prior art before building a new
feature.

## Feature Index

| Feature | Status | Owner | Related Docs |
| --- | --- | --- | --- |
| Example: Documentation starter | Example | Project maintainers | `docs/CHANGELOG.md`, `docs/DEPENDENCIES.md`, `docs/TESTING.md` |

## Example Feature Entries

The entries in this section are examples for this starter template. When copying
this setup into a real project, replace them with that project's actual
features, systems, workflows, or mechanics.

### Example Feature: Documentation Starter

- Status: Example
- Summary: Provides a reusable structure for Codex project memory.
- User value: Helps Codex and contributors preserve decisions, dependencies,
  testing expectations, and security notes.
- Main files or systems:
  - `AGENTS.md`
  - `docs/`
- Dependencies:
  - None
- Testing notes:
  - Verify referenced documentation files exist.
- Change history:
  - Example entry included with the starter repository.

## Optional Game-Development Notes

For games, document mechanics and asset rules as features. Examples:

- Player movement system
- Enemy spawning rules
- Inventory behavior
- Dialogue system
- Spritesheet animation reader
- Save-game format

When a mechanic depends on an asset convention, record that relationship in
`docs/DEPENDENCIES.md` and the asset details in `docs/ASSETS.md`.
